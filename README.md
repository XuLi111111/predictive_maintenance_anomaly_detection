# Predictive Maintenance — Anomaly Detection for Industrial Pumps

**CITS5206 Capstone Project · Group 14 · University of Western Australia**

A short-horizon predictive maintenance system for industrial centrifugal
pumps. Given the last 20 seconds of 8-channel sensor data, the system
predicts whether an anomaly will occur within the next 10 seconds and
surfaces a four-tier alert state in a live web dashboard. Industry
partner **Programmed**, sponsor **Peter Whittaker** (Mechanical
Engineering, UWA).

The trained pipeline reaches **F1 = 0.9244** on the SKAB held-out test
split (TransformerFusionLite, selected from a 54-config sweep).

## End-to-end workflow

The repository ships **no** model weights, scalers, or datasets — only
the code that produces them. A fresh clone reproduces the entire system
in four steps.

### Step 1 · Get the SKAB dataset

The Skoltech Anomaly Benchmark is a public open dataset. Place its CSV
files under `data/raw/skab/` so the layout looks like:

```
data/raw/skab/
├── valve1/           0.csv ~ 15.csv     (16 experiment runs)
├── valve2/           0.csv ~ 3.csv      ( 4 experiment runs — test split)
├── anomaly-free/     anomaly-free.csv
└── other/            9.csv ~ 23.csv     (operational variations)
```

Source: <https://github.com/waico/SkAB>.

### Step 2 · Install training-pipeline dependencies

```bash
pip install -r requirements.txt
```

This pulls in `numpy`, `pandas`, `scikit-learn`, `xgboost`, `torch`,
`joblib`, `matplotlib` — everything the three training scripts need.

### Step 3 · Run the training pipeline

```bash
# 3.1  Build the sliding-window dataset (writes
#      data/processed/skab/skab_window20_horizon10.npz)
python src/data_preprocessing/Build_Dataset_SKAB_DLpipeline_By_David.py

# 3.2  Train the 7 classical ML models + scaler (writes
#      app/backend/artifacts/{model_*.pkl, scaler.pkl} directly)
python src/classical_ml/SKAB_ClassicalML_Baseline_By_David.py

# 3.3  Train the TransformerFusionLite sweep (writes
#      app/backend/artifacts/{model_transformer.pt, transformer_scaler.pkl,
#      transformer_threshold.json} directly; per-config sweep intermediates
#      go to results/skab/transformer/)
python src/deep_learning/SKAB_TransformerFusionLite_TrainingSearch_ByXuLi.py
```

The training scripts deposit every inference-ready artifact straight
into `app/backend/artifacts/` — the same directory the FastAPI worker
reads from at runtime — so the next step needs no manual copy.

**Optional diagnostics**

```bash
# Audit the SKAB CSVs for timestamp overlaps and label conflicts.
python -c "from src.data_preprocessing.SKAB_DataQualityAudit_ForDLpipeline_ByXuLi import audit_valve_only; audit_valve_only()"

# Regenerate the EDA figures under docs/eda/.
python src/data_preprocessing/SKAB_EDA_ByXuLi.py
```

### Step 4 · Run the web application

```bash
cd app
docker compose up --build
```

- UI:        <http://localhost:8080>
- Live page: <http://localhost:8080/live>
- API docs:  <http://localhost:8080/api/docs>

To exercise the Live page, push simulated samples from a second terminal:

```bash
python app/scripts/pump_simulator.py
```

See [`app/README.md`](app/README.md) for the full app-level setup, API
reference, bare-metal (non-Docker) instructions, and pytest invocation.

## Technical approach

- **Labelling**: future-horizon binary label — `1` if any anomaly occurs
  in the next `HORIZON=10` seconds, else `0`.
- **Windowing**: sliding window of `WINDOW_SIZE=20` past time steps,
  stride 1.
- **Standardisation**: per-`(timestep × feature)` `StandardScaler` for
  classical models; per-feature `StandardScaler` for the transformer
  (the inference layer keeps the two scalers separate).
- **Class imbalance**: `pos_weight` in `BCEWithLogitsLoss` (transformer)
  + no resampling for the classical baselines.
- **Inference**: per-window stateless scoring; the live monitor adds a
  four-tier alert state machine (NORMAL / WATCH / WARNING / ALERT) with
  promotion + demotion hysteresis.

See [`SKAB_Preprocessing_README.md`](SKAB_Preprocessing_README.md) for
the labelling and split logic in detail.
